# 国赛 vs 美赛核心差异说明

本文档梳理全国大学生数学建模竞赛（CUMCM）与美国大学生数学建模竞赛（MCM/ICM）在建模和论文环节的核心差异。在 Skill 执行建模任务时，需根据用户参加的竞赛类型自动调整策略和侧重点。

---

## 速查表：国赛 vs 美赛核心差异

| 对比维度               | 国赛（CUMCM）                                     | 美赛（MCM/ICM）                                   |
| ---------------------- | ------------------------------------------------- | ------------------------------------------------- |
| **建模侧重**     | 模型与问题的匹配度、结果正确性与实用性            | 创新性（评分最高权重）、建模过程完整性            |
| **模型选择**     | 优先成熟模型，鼓励分层改进（基础→改进→优化）    | 鼓励原创模型或突破性改进，可融合AI/ML等非传统方法 |
| **检验要求**     | 灵敏度分析 + 误差分析 + 稳定性分析缺一不可        | 不确定性分析 + 多情景分析 + 鲁棒性评估            |
| **数据要求**     | 强调数据真实性与预处理可追溯，禁止造假            | 不强求真实数据，合理假设亦可                      |
| **论文格式**     | 严格硬性约束（页边距、页码、≤20页、摘要≤300字） | 相对宽松（20-30页，LaTeX/Word均可）               |
| **摘要/Summary** | 摘要含问题、方法、核心结果、结论，必须有具体数据  | Summary需含背景→方法→结果→意义，突出创新点     |
| **语言风格**     | 简洁精准，段落式自然陈述，拒绝死板罗列            | 故事化表达，强调可读性和沟通流畅性                |
| **图表要求**     | 规范清晰，必须有标题、坐标轴标签、单位            | 专业美观，需与核心结论关联，有逻辑性              |
| **附录**         | 源代码、数据文件、详细计算过程必须包含            | 不强求完整代码，但建议包含关键算法                |
| **参考文献**     | GB/T 7714格式，引用3-5篇高质量文献                | APA/MLA格式统一即可，引用所有外部成果             |
| **关键底线**     | 格式硬约束违规直接降档                            | 学术不端（plagiarism）零容忍                      |

## 一、国赛（CUMCM）要点详解

### 1.1 建模环节

#### 必须重视

1. **问题与模型的匹配度**

   - 拒绝"为复杂而复杂"，优先选择成熟、已被验证的模型适配实际问题
   - 每个模型的选择必须有明确理由，不能堆砌算法
   - 示例：能用线性回归解决的就不要上神经网络
2. **数据真实性与预处理**

   - 缺失值、异常值处理过程需完整记录，保证可追溯
   - 数据来源必须注明，禁止伪造数据
   - 预处理步骤：缺失值处理 → 异常值检测 → 标准化/归一化
3. **模型严谨性**

   - 公式推导简洁规范，符号定义统一无歧义
   - 所有符号必须在首次出现时定义，建议建立术语表
4. **结果正确性与实用性**

   - 数值需贴合实际场景，误差在合理范围内
   - 结论需回应题目提出的具体问题
5. **假设合理性**

   - 每条假设必须紧扣问题，且有明确依据
   - 关键约束条件不能随意简化

#### 强烈鼓励

1. **分层建模**：基础模型 → 改进模型 → 优化模型，体现层次深度
2. **模型改进创新**：对经典模型做针对性调整，说明改进的价值和原因
3. **多方法对比**：用简单模型作为基准，衬托改进模型的优势
4. **完整检验**：灵敏度分析、误差分析、稳定性分析一个不能少
5. **模型推广性**：思考模型在同类问题中的复用价值

#### 核心避坑

- ❌ 过度追求高深模型，忽略问题本质（如用深度学习解决线性可解问题）
- ❌ 跳过模型检验，直接下结论
- ❌ 假设与问题脱节，忽略关键约束条件
- ❌ 数据来源不注明或引用不可靠数据
- ❌ 求解过程无复现性，缺少关键参数和计算步骤

### 1.2 论文环节

#### 必须重视

1. **格式硬性规范**

   - 页边距标准、页码正确、摘要字数≤300字、无目录、正文≤20页
   - 格式违规直接降档，属于底线要求
2. **摘要质量**

   - 必须包含：问题背景 → 所用方法 → 核心结果（具体数据） → 结论
   - 拒绝空泛描述，如"建立了模型，取得了较好效果"这种无效表述
3. **结构完整性**

   - 标准章节：摘要 → 问题重述 → 模型假设 → 符号说明 → 模型建立与求解 → 模型检验与误差分析 → 灵敏度分析 → 模型评价与推广 → 参考文献 → 附录
4. **图表规范性**

   - 每个图表必须有标题、坐标轴标签、单位
   - 分辨率清晰，曲线可区分
5. **附录完整性**

   - 源代码、数据文件、详细计算过程必须包含
   - 遗漏会导致无法复现，直接扣分

#### 强烈鼓励

1. **语言简洁精准**：无冗余描述，每段紧扣核心观点
2. **结果可视化**：用图表替代大段文字，关键数据一目了然
3. **参考文献规范**：GB/T 7714格式，引用3-5篇高质量文献
4. **问题重述精炼**：不复制题目原文，用自己的语言提炼核心矛盾
5. **模型局限性分析**：客观说明模型的适用范围和不足

#### 核心避坑

- ❌ 格式错误（页码混乱、页边距不足、摘要超字数）
- ❌ 摘要空泛（无具体方法、无数据支撑）
- ❌ 图表无说明（直接贴图不解释其反映的结论）
- ❌ 符号不统一（同一符号表示不同含义或前后不一致）
- ❌ 附录缺失（漏传源代码导致结果无法复现）
- ❌ 参考文献格式混乱

## 二、美赛（MCM/ICM）要点详解

### 2.1 建模环节

#### 必须重视

1. **创新性**

   - 这是美赛评分权重最高的维度
   - 拒绝简单套用现有模型，要有独创性思考
   - 对经典模型的改进必须是实质性的，而非表面修改
2. **建模过程完整性**

   - 逻辑链条无断层：问题分析 → 假设 → 建模 → 求解 → 验证 → 拓展
   - 每个步骤都需详细阐述，评委需要能完全理解你的思路
3. **假设论证**

   - 每个假设都需要充分论证其合理性
   - 最好能用数据或文献支撑假设
   - 不可凭空设定假设
4. **跨学科视角**

   - 尤其ICM题，需结合社会、伦理、政策等多维度
   - MCM题也鼓励多角度思考
5. **不确定性分析**

   - 评估模型对参数变化、假设偏差的鲁棒性
   - 说明模型在什么条件下成立、什么条件下可能失效

#### 强烈鼓励

1. **原创模型或突破性改进**

   - 鼓励融合AI、机器学习、大数据等非传统方法
   - 将不同领域的模型跨界组合
2. **多情景分析**

   - 考虑不同约束条件、不同参数取值下的模型表现
   - 对比分析不同情景下的结果差异
3. **模型推广性**

   - 说明模型在其他场景的应用潜力
   - 展示模型的通用价值
4. **定性与定量结合**

   - ICM题需兼顾文字论证与数据支撑
   - MCM题中定性分析可补充定量分析的不足
5. **团队协作体现**

   - 论文中可简要说明分工，展示团队协作逻辑

#### 核心避坑

- ❌ 中式思维——只重结果忽略过程详述
- ❌ 假设无依据——未论证假设的合理性或假设脱离现实
- ❌ 建模过程断层——跳过关键推导步骤，评委无法理解逻辑
- ❌ 忽视ICM/MCM差异——ICM缺少社会/伦理维度分析，MCM缺少量化分析
- ❌ 学术不端——引用他人成果未注明，plagiarism零容忍
- ❌ 只谈优点不谈缺点——未分析模型局限性

### 2.2 论文环节

#### 必须重视

1. **Summary质量**

   - 这是美赛论文的"定海神针"，决定第一印象和基础分
   - 必须包含：背景 → 问题 → 方法（含创新点） → 核心结果 → 意义
   - 必须有具体数据和创新点，空泛的Summary直接导致低分
2. **沟通表达流畅性**

   - 英文要地道、自然，评委能轻松理解建模思路
   - 避免中式翻译和生硬的学术腔
3. **结构完整性**

   - 标准章节：Summary → Introduction → Assumptions → Model → Solution → Analysis → Conclusion
   - 每个章节之间有自然的过渡
4. **图表专业性**

   - 美观清晰，图文呼应
   - 图表之间应有逻辑关联，共同服务于核心结论
5. **引用规范**

   - 所有外部成果必须注明来源
   - 引用格式统一（APA或MLA均可，但全文一致）

#### 强烈鼓励

1. **故事化表达**

   - 用"问题 → 挑战 → 方案 → 效果"的逻辑串联全文
   - 增强可读性，让评委有阅读兴趣
2. **详细的Assumptions Justifications**

   - 用数据或文献支撑假设的合理性
   - 包含敏感性测试说明假设的影响
3. **深入的Analysis**

   - 不仅谈"结果是多少"，更要分析"为什么是这个结果""如何改进"
   - 展示团队的批判性思维能力
4. **专业排版**

   - 优先使用LaTeX，配合美赛官方模板
   - 目录、公式、图表排版整齐
5. **未来研究方向**

   - 在Conclusion中提出可延伸的研究点
   - 体现对问题理解的深度

#### 核心避坑

- ❌ Summary空洞——无具体数据、无创新点
- ❌ 英文表达问题——语法/拼写错误、中式翻译、专业术语误用
- ❌ 建模过程简化——跳过关键步骤导致逻辑断裂
- ❌ 图表无关联——图表与文字脱节或缺少标题/说明
- ❌ 参考文献缺失或格式混乱
- ❌ 页数失控——过度冗余或过短（建议20-30页）

## 三、美赛MCM与ICM的内部差异

| 维度               | MCM（A/B题）                   | ICM（C/D/E/F题）                    |
| ------------------ | ------------------------------ | ----------------------------------- |
| **题型特征** | 连续型/离散型，偏数学和工程    | 数据洞察/运筹/环境/政策，偏综合应用 |
| **建模风格** | 量化分析为主，强调数学严谨性   | 定性+定量结合，强调跨学科视角       |
| **论文风格** | 技术报告风格，重视公式和算法   | 混合风格，需兼顾社会/伦理/政策维度  |
| **数据使用** | 题目通常提供数据，基于数据建模 | 可能需要自主收集或合理假设数据      |
| **创新方向** | 算法创新、模型结构创新         | 方法论创新、视角创新、融合创新      |

## 四、Skill行为适配指南

当 Skill 执行建模任务时，需要根据竞赛类型自动调整以下行为：

### 4.1 竞赛类型识别

通过用户输入自动判断竞赛类型：

- 关键词"国赛""CUMCM""全国大学生数学建模" → **国赛模式**
- 关键词"美赛""MCM""ICM""美国大学生数学建模" → **美赛模式**
- 未明确指定时，主动询问用户参加的是国赛还是美赛

### 4.2 建模阶段适配

| 行为                   | 国赛模式                                                           | 美赛模式                                                    |
| ---------------------- | ------------------------------------------------------------------ | ----------------------------------------------------------- |
| **模型选择策略** | 优先推荐成熟、经典的模型；如有改进必须分层展示（基础→改进→优化） | 鼓励创新组合；可推荐非传统方法（AI/ML等）；重点说明创新点   |
| **假设处理**     | 简洁说明依据即可                                                   | 每个假设需要详细论证（Justification），最好有文献或数据支撑 |
| **检验要求**     | 必须包含灵敏度分析、误差分析、稳定性分析三部分                     | 必须包含不确定性分析、敏感性测试、多情景对比                |
| **结果呈现**     | 强调精确性和实用性，结果需回应题目具体问题                         | 强调解释性，不仅说结果是什么，更要分析为什么、如何改进      |

### 4.3 论文阶段适配

| 行为                   | 国赛模式                               | 美赛模式                                                |
| ---------------------- | -------------------------------------- | ------------------------------------------------------- |
| **摘要/Summary** | 摘要≤300字，含问题+方法+核心数据+结论 | Summary含背景+问题+方法（创新点）+结果+意义，有具体数据 |
| **写作风格**     | 简洁、精炼、直接，段落式自然陈述       | 故事化、有条理、逻辑清晰，重视可读性和表达流畅性        |
| **语言**         | 中文                                   | 英文（地道表达，避免中式翻译）                          |
| **格式约束**     | 严格执行页边距/页码/页数限制/无目录    | 相对宽松，建议LaTeX排版，20-30页                        |
| **参考文献**     | GB/T 7714中文格式                      | APA或MLA格式（全文统一）                                |
| **附录**         | 必须包含完整源代码和数据               | 建议包含关键算法，不强求完整代码                        |
| **图表**         | 规范清晰，有标题和坐标轴标注           | 专业美观，图文呼应，有设计感                            |
| **模型评价**     | 客观说明局限性和适用范围               | 深入分析改进方向，提出未来研究展望                      |

### 4.4 ICM专属适配

当识别到美赛ICM（C/D/E/F题）时，额外启用：

- 在建模中引入社会/伦理/政策/环境维度分析
- 定性分析与定量分析并重
- 论文中增加跨学科讨论段落
- 鼓励结合真实世界案例和背景知识

## 五、关键提醒

1. **国赛底线**：格式硬约束是底线，违规直接降档——摘要是生命线，格式是及格线。
2. **美赛天花板**：创新+沟通是上限，决定能否冲击O奖/F奖——Summary定生死，创新定高度。
3. **共同底线**：学术不端（数据造假、抄袭、未注明引用）一旦触碰，国赛美赛均直接取消成绩。
4. **共同逻辑**：无论国赛美赛，核心流程"问题→假设→模型→求解→检验→结论"是基础，国赛重在"深度"（结果精准度），美赛重在"广度"（过程完整性和创新性）。
5. **时间分配建议**：
   - 国赛（3天72小时）：建模18h + 编程18h + 论文28h + 缓冲8h
   - 美赛（4天96小时）：建模24h + 编程24h + 论文38h + 缓冲10h
