# 数学建模技能 - Math Modeling

本技能为数学建模竞赛提供**三角色 + 一指导**协作模式的三阶段工作流程：建模分析 → 代码实现 → 论文撰写，阶段切换时由指导教师质量门禁审查。

## ⚠️ 竞赛类型识别（最优先执行）

在执行任何任务前，必须先确认竞赛类型：

- 用户提及 **"国赛""CUMCM""全国大学生数学建模"** → 国赛模式
- 用户提及 **"美赛""MCM""ICM""美国大学生数学建模"** → 美赛模式
- 用户未明确指定 → **主动询问**

> 国赛和美赛在建模策略、检验要求、论文格式、评审权重上存在根本差异。**必须先读取 `references/国赛美赛差异说明.md`**。

## 工作流程

```
┌──────────────────────────────────────────────────────────────────────────┐
│  建模分析            代码实现            论文撰写                         │
│  (建模手)    ──→    (编程手)    ──→    (论文手)                          │
│       │   审查通过        │   审查通过        │  终审通过                  │
│       ▼                  ▼                  ▼                            │
│  ┌─────────┐       ┌─────────┐       ┌─────────┐                        │
│  │ 指导教师 │       │ 指导教师 │       │ 指导教师 │                        │
│  │ 思路启发 │       │ 建模审查 │       │ 结果检查 │                        │
│  │ 避坑提醒 │       │ 编码提醒 │       │ 写作提醒 │                        │
│  └─────────┘       └─────────┘       └─────────┘                        │
│       ↑                  ↑                  ↑          ┌─────────┐      │
│       │                  │                  │          │ 指导教师 │      │
│       │                  │                  │          │ 终稿审查 │      │
│       └──────────────────┴──────────────────┴──────────┴─────────┘      │
│                          全程答疑                                         │
└──────────────────────────────────────────────────────────────────────────┘
```

| 角色 | 定位 | 阶段 | 核心职责 |
|:----:|------|------|---------|
| **建模手** | 大脑 | 第一阶段 | 题目分析、模型选择、算法设计、术语规范 |
| **编程手** | 双手 | 第二阶段 | 代码编写、模型求解、数据可视化 |
| **论文手** | 喉舌 | 第三阶段 | 论文撰写、成果整理、摘要打磨 |
| **指导教师** | 舵手 | 全程 + 门禁 | 方向把关、阶段审查、终审、答疑 |

## 执行顺序

### 第零步：赛前准备
读取 `references/roles/指导教师说明.md`，执行"时机1"——思路启发与避坑提醒。

### 第一步：建模分析（建模手）
**先读取** `references/roles/建模手说明.md` 和 `references/国赛美赛差异说明.md`。
**产出**：`题目分析报告.md` + `术语表格.md`。不写代码。
**门禁**：读取 `references/roles/指导教师说明.md` 执行"时机2"审查。通过后方可进入下一阶段。

### 第二步：代码实现（编程手）
**先读取** `references/roles/编程手说明.md`。
**产出**：求解代码 + `结果表格.csv` + 可视化图表 + `README.md`。
**门禁**：读取 `references/roles/指导教师说明.md` 执行"时机3"审查。通过后方可进入下一阶段。

### 第三步：论文撰写（论文手）
**先读取** `references/roles/论文手说明.md` + 对应模板（`references/国赛论文模板.md` 或 `references/美赛论文模板.md`）。
**产出**：`论文.md`（Markdown 格式）。
**注意**：写完后用总时间 20% 打磨摘要/Summary，对照反AI味检查清单自查。
**终审**：读取 `references/roles/指导教师说明.md` 执行"时机4"终审。通过后论文完成。

## 关键原则

1. **竞赛类型决定一切**：国赛和美赛策略不同，决不混淆（先读差异说明）
2. **严格按序执行**：建模 → 编程 → 论文，每次阶段切换必须通过指导教师门禁审查
3. **术语统一**：建模阶段创建的术语表，后续阶段必须沿用
4. **必写核心不可省**：论文必须含模型检验和灵敏度分析（国赛缺一不可）
5. **摘要决定生死**：写完正文后回头打磨摘要/Summary，占总时间 20%
6. **避免AI味**：论文完成后用 `references/roles/论文手说明.md` 中的反AI味检查清单逐项自查
